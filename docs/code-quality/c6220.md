# <a name="c6220"></a>C6220
avviso C6220-cast implicito tra tipi integer semanticamente diversi: confronto di HRESULT con-1. Prendere in considerazione l'uso della macro SUCCEEDed o FAILED

 Questo avviso indica che è in corso il confronto di un `HRESULT` con un valore esplicito non `HRESULT`, che non è un formato `HRESULT` valido. Un errore in `HRESULT` (`E_FAIL`) non è rappresentato da-1. Pertanto, un cast implicito di un `HRESULT` a un Integer genererà un valore non corretto e probabilmente condurrà il risultato errato.

## <a name="example"></a>Esempio
 Nella maggior parte dei casi, questo avviso è causato dal fatto che il codice si aspetta in modo errato che una funzione che deve restituire un `HRESULT` restituisce invece un Integer, usando-1 come valore di errore. L'esempio di codice seguente genera questo avviso:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == -1)
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 È consigliabile usare la macro `SUCCEEDED` o `FAILED` per verificare il valore di un `HRESULT`. Per correggere il problema, usare il codice seguente:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Per questo avviso, il tipo `SCODE` equivale a `HRESULT`.

 Il confronto esplicito è appropriato per verificare la presenza di valori `HRESULT` specifici, ad esempio `E_FAIL`. In caso contrario, utilizzare le macro `SUCCEEDED` o `FAILED`.

 Per ulteriori informazioni, vedere [macro SUCCEEDED](http://go.microsoft.com/fwlink/?LinkId=92738) e [macro failed](http://go.microsoft.com/fwlink/?LinkId=180875).

 Si noti che l'uso di malloc e della versione gratuita (e delle API di allocazione della memoria dinamica correlate) presenta molti problemi in termini di perdite di memoria ed eccezioni. Per evitare completamente questi tipi di perdite e problemi di eccezione, utilizzare i meccanismi forniti dalla libreria STL ( C++ Standard Template Library). Sono inclusi [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)e [vector](/cpp/standard-library/vector). Per altre informazioni, vedere [puntatori intelligenti](/cpp/cpp/smart-pointers-modern-cpp) e [ C++ libreria standard](/cpp/standard-library/cpp-standard-library-reference).
