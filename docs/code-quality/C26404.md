# <a name="c26404--dontdeleteinvalid"></a>C26404 DONT_DELETE_INVALID
Una volta puntatore proprietario rilascia o trasferisce la propria risorsa, ottiene in uno stato "non valido". Eliminazione di un puntatore di questo tipo pu√≤ causare danneggiamento della memoria immediato a causa dell'eliminazione double o una violazione di accesso quando si accede alla risorsa eliminata da un altro puntatore proprietario.

## <a name="example-1-deleting-an-owner-after-transferring-its-value"></a>Esempio 1: Eliminazione di un proprietario dopo il trasferimento, il relativo valore
```cpp
gsl::owner<State*> validState = nullptr;
gsl::owner<State*> state = ReadState();
validState = state;
if (!IsValid(state))
    delete state;   // C26404
```

## <a name="example-2-deleting-an-uninitialized-owner"></a>Esempio 2: Eliminazione di un proprietario non inizializzato
```cpp
gsl::owner<Message*> message;
if (popLast)
    message = ReleaseMessage();
delete message; // C26404
```
