# <a name="ca1831-use-asspan-instead-of-range-based-indexers-for-string-when-appropriate"></a>CA1831: Usare AsSpan invece di indicizzatori basati su Range per la stringa quando appropriato

|Elemento|valore|
|-|-|
|CheckId|Ca1831|
|Category|Microsoft. performance|
|Modifica|Senza interruzioni|

## <a name="cause"></a>Causa

Un indicizzatore di intervallo viene usato in una stringa e il valore viene assegnato in modo implicito a `ReadOnlySpan<char>` .

## <a name="rule-description"></a>Descrizione della regola

Questa regola viene attivata quando si usa un indicizzatore di intervallo in una stringa e lo si assegna a un tipo span. L'indicizzatore di intervallo in un oggetto <xref:System.Span%601> è un'operazione non di copia <xref:System.Span%601.Slice%2A?#System_Span_1_Slice_System_Int32_System_Int32_> , ma per l'indicizzatore di intervallo in una stringa <xref:System.String.Substring%2A?#System_String_Substring_System_Int32_System_Int32_> verrà usato il metodo anziché <xref:System.Span%601.Slice%2A?#System_Span_1_Slice_System_Int32_System_Int32_> . Viene prodotta una copia della parte richiesta della stringa. Questa copia in genere non è necessaria quando viene usata in modo implicito come <xref:System.ReadOnlySpan%601> <xref:System.ReadOnlyMemory%601> valore o. Se non è prevista una copia, usare il <xref:System.MemoryExtensions.AsSpan%2A?#System_MemoryExtensions_AsSpan_System_String_AsSpan> metodo per evitare la copia non necessaria. Se la copia è progettata, assegnarla prima di tutto a una variabile locale o aggiungere un cast esplicito. L'analizzatore segnala solo quando viene usato un cast implicito sul risultato dell'operazione dell'indicizzatore di intervallo.

### <a name="detects"></a>Rileva

Conversione implicita:

`ReadOnlySpan<char> slice = str[a..b];`

### <a name="does-not-detect"></a>Non rileva

Conversione esplicita:

`ReadOnlySpan<char> slice = (ReadOnlySpan<char>)str[a..b];`

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

Per correggere una violazione di questa regola, usare <xref:System.MemoryExtensions.AsSpan%2A?#System_MemoryExtensions_AsSpan_System_String_AsSpan> anziché l' <xref:System.Range> indicizzatore basato su sulla stringa per evitare la creazione di copie di dati non necessarie.

```csharp
public void TestMethod(string str)
{
    // The violation occurs
    ReadOnlySpan<char> slice = str[1..3];
    ...
}
```

```csharp
public void TestMethod(string str)
{
    // The violation fixed with AsSpan extension method
    ReadOnlySpan<char> slice = str.AsSpan()[1..3];
    ...
}
```

> [!TIP]
> Una correzione del codice è disponibile per questa regola in Visual Studio. Per usarlo, posizionare il cursore sulla violazione della matrice e premere **CTRL** + **.** (punto). Scegliere **USA AsSpan anziché l'indicizzatore basato su intervallo in una stringa** dall'elenco di opzioni presentate.
>

## <a name="when-to-suppress-warnings"></a>Quando escludere gli avvisi

È possibile eliminare una violazione di questa regola se è prevista la creazione di una copia. Oltre ai [consueti modi per eliminare gli avvisi di analisi del codice](in-source-suppression-overview.md), è anche possibile aggiungere un cast esplicito per evitare l'avviso.

```csharp
public void TestMethod(string str)
{
    // The violation occurs.
    ReadOnlySpan<char> slice = str[1..3];
    ...
}
```

```csharp
public void TestMethod(string str)
{
    // The violation avoided with explicit casting.
    ReadOnlySpan<char> slice = (ReadOnlySpan<char>)str[1..3];
    ...
}
```
