# <a name="ca5370-use-xmlreader-for-validating-reader"></a>CA5370: Usare XmlReader per la convalida del lettore

|Elemento|valore|
|-|-|
|CheckId|CA5370|
|Category|Microsoft.Security|
|Modifica|Senza interruzioni|

## <a name="cause"></a>Causa

La convalida di input XML non attendibile con la classe di cui è stata `XmlValidatingReader` creata un'istanza senza un `XmlReader` oggetto può causare un attacco Denial of Service, la divulgazione di informazioni e la richiesta lato server falsa. Questi attacchi sono abilitati dalla DTD non attendibile e dall'elaborazione XML Schema, che consente l'inclusione di bombe XML e entità esterne dannose nel codice XML. Solo con `XmlReader` è possibile disabilitare la DTD. L'elaborazione inline XML Schema come `XmlReader` ha `ProhibitDtd` la `ProcessInlineSchema` proprietà e impostata su per `false` impostazione predefinita in .NET Framework a partire dalla versione 4,0.

## <a name="rule-description"></a>Descrizione della regola

L'elaborazione di schemi DTD e XML non attendibili può consentire il caricamento di riferimenti esterni pericolosi. Questo caricamento pericoloso può essere limitato utilizzando un `XmlReader` con un resolver sicuro oppure con la DTD e l'elaborazione dello schema inline XML disabilitata. Questa regola rileva il codice che usa la `XmlValidatingReader` classe senza `XmlReader` come parametro del costruttore.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

- Utilizzare `XmlValidatingReader(XmlReader)` con `ProhibitDtd` le `ProcessInlineSchema` proprietà e impostate su `false` .
- A partire da .NET Framework 2,0, `XmlValidatingReader` viene considerato obsoleto. È possibile creare un'istanza di un reader di convalida con <xref:System.Xml.XmlReader.Create%2A?displayProperty=nameWithType> .

## <a name="when-to-suppress-warnings"></a>Quando escludere gli avvisi

Questo avviso può essere eliminato se `XmlValidatingReader` viene sempre usato per convalidare il codice XML che deriva da un'origine attendibile e pertanto non può essere alterato.

## <a name="pseudo-code-examples"></a>Esempi di pseudo-codice

### <a name="violation"></a>Violazione

Nell'esempio di pseudo-codice seguente viene illustrato il modello rilevato da questa regola.
Il tipo del primo parametro di `XmlValidatingReader.XmlValidatingReader()` non è `XmlReader` .

```csharp
using System;
using System.IO;
using System.Xml;
...
public void TestMethod(Stream xmlFragment, XmlNodeType fragType, XmlParserContext context)
{
    var obj = new XmlValidatingReader(xmlFragment, fragType, context);
}
```

### <a name="solution"></a>Soluzione

```csharp
using System;
using System.Xml;
...
public void TestMethod(XmlReader xmlReader)
{
    var obj = new XmlValidatingReader(xmlReader);
}
```
