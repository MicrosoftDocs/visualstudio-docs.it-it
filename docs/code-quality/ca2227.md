# <a name="ca2227-collection-properties-should-be-read-only"></a>CA2227: Le proprietà di raccolte devono essere in sola lettura

|||
|-|-|
|TypeName|CollectionPropertiesShouldBeReadOnly|
|CheckId|CA2227|
|Category|Microsoft.Usage (Utilizzo)|
|Modifica|Interruzione|

## <a name="cause"></a>Causa

Una proprietà scrivibile visibile esternamente è di <xref:System.Collections.ICollection?displayProperty=fullName>un tipo che implementa . Questa regola ignora le matrici, gli indicizzatori (proprietà con il nome 'Item'), le raccolte non modificabili, le raccolte di sola lettura e i set di autorizzazioni.

## <a name="rule-description"></a>Descrizione della regola

Una proprietà di raccolta scrivibile consente a un utente di sostituire la raccolta con una raccolta completamente diversa. Una proprietà di sola lettura interrompe la sostituzione della raccolta, ma consente comunque l'impostazione dei singoli membri. Se la sostituzione della raccolta è un obiettivo, il modello di progettazione preferito consiste nell'includere un metodo per rimuovere tutti gli elementi dalla raccolta e un metodo per ripopolare la raccolta. Vedere <xref:System.Collections.ArrayList.Clear%2A> i <xref:System.Collections.ArrayList.AddRange%2A> metodi <xref:System.Collections.ArrayList?displayProperty=fullName> e della classe per un esempio di questo modello.

Sia la serializzazione binaria che quella XML supportano proprietà di sola lettura che sono raccolte. La <xref:System.Xml.Serialization.XmlSerializer?displayProperty=fullName> classe ha requisiti specifici <xref:System.Collections.ICollection> <xref:System.Collections.IEnumerable?displayProperty=fullName> per i tipi che implementano e per essere serializzabili.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

Per correggere una violazione di questa regola, rendere la proprietà di sola lettura. Se la progettazione lo richiede, aggiungere metodi per cancellare e ripopolare la raccolta.

## <a name="when-to-suppress-warnings"></a>Quando eliminare gli avvisi

È possibile eliminare l'avviso se la proprietà fa parte di una classe [DTO (Data Transfer Object).](/previous-versions/msp-n-p/ff649585(v=pandp.10))

In caso contrario, non eliminare gli avvisi da questa regola.
