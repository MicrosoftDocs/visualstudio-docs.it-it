# <a name="ca1811-avoid-uncalled-private-code"></a>CA1811: Evitare il codice privato non chiamato

|||
|-|-|
|TypeName|AvoidUncalledPrivateCode|
|CheckId|CA1811|
|Category|Microsoft.Performance|
|Modifica|Non-rottura|

## <a name="cause"></a>Causa
Un membro privato o interno (a livello di assembly) non dispone di chiamanti nell'assembly, non viene richiamato da Common Language Runtime e non viene richiamato da un delegato. I seguenti membri non vengono controllati in base a questa regola:

- Membri di interfaccia espliciti.

- Costruttori statici.

- Costruttori di serializzazione.

- Metodi contrassegnati con <xref:System.Runtime.InteropServices.ComRegisterFunctionAttribute?displayProperty=fullName> o <xref:System.Runtime.InteropServices.ComUnregisterFunctionAttribute?displayProperty=fullName>.

- Membri con override.

## <a name="rule-description"></a>Descrizione della regola
Questa regola può segnalare falsi positivi se si verificano punti di ingresso che non sono attualmente identificati dalla logica della regola. Inoltre, un compilatore può generare codice non richiamabile in un assembly.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni
Per correggere una violazione di questa regola, rimuovere il codice non richiamabile o aggiungere il codice che la chiama.

## <a name="when-to-suppress-warnings"></a>Quando eliminare gli avvisi
È sicuro escludere un avviso da questa regola. A tale scopo, consultate [Eliminazione degli avvisi dell'analisi](../code-quality/in-source-suppression-overview.md)del codice.
