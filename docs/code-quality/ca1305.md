# <a name="ca1305-specify-iformatprovider"></a>CA1305: Specificare IFormatProvider

|||
|-|-|
|TypeName|SpecifyIFormatProvider|
|CheckId|CA1305|
|Category|Microsoft.Globalization (Globalizzazione)|
|Modifica|Non-rottura|

## <a name="cause"></a>Causa

Un metodo o un costruttore chiama uno o <xref:System.IFormatProvider?displayProperty=fullName> più membri che dispongono di overload che <xref:System.IFormatProvider> accettano un parametro e il metodo o costruttore non chiama l'overload che accetta il parametro.

Questa regola ignora le chiamate ai metodi .NET <xref:System.IFormatProvider> documentate come ignorando il parametro. La regola ignora anche i seguenti metodi:

- <xref:System.Activator.CreateInstance%2A?displayProperty=nameWithType>
- <xref:System.Resources.ResourceManager.GetObject%2A?displayProperty=nameWithType>
- <xref:System.Resources.ResourceManager.GetString%2A?displayProperty=nameWithType>
- <xref:System.Boolean.ToString%2A?displayProperty=nameWithType>
- <xref:System.Char.ToString%2A?displayProperty=nameWithType>
- <xref:System.Guid.ToString%2A?displayProperty=nameWithType>

## <a name="rule-description"></a>Descrizione della regola

Quando <xref:System.Globalization.CultureInfo?displayProperty=nameWithType> non <xref:System.IFormatProvider> viene fornito un oggetto o , il valore predefinito fornito dal membro di overload potrebbe non avere l'effetto desiderato in tutte le impostazioni locali. Inoltre, i membri .NET scelgono le impostazioni cultura e la formattazione predefinite in base a presupposti che potrebbero non essere corretti per il codice. Per assicurarsi che il codice funzioni come previsto per gli scenari, è necessario fornire informazioni specifiche delle impostazioni cultura in base alle linee guida seguenti:To make sure that the code works as expected for your scenarios, you should fornire culture-specific information according to the following guidelines:

- Se il valore verrà visualizzato all'utente, utilizzare le impostazioni cultura correnti. Vedere <xref:System.Globalization.CultureInfo.CurrentCulture%2A?displayProperty=nameWithType>.

- Se il valore verrà archiviato e accessibile dal software (salvato in un file o in un database), utilizzare le impostazioni cultura invarianti. Vedere <xref:System.Globalization.CultureInfo.InvariantCulture%2A?displayProperty=nameWithType>.

- Se non si conosce la destinazione del valore, fare in modo che il consumer di dati o il provider specifichi le impostazioni cultura.

Anche se il comportamento predefinito del membro di overload è appropriato per le proprie esigenze, è preferibile chiamare in modo esplicito l'overload specifico delle impostazioni cultura in modo che il codice si autodocumenta e più facilmente gestito.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

Per correggere una violazione di questa regola, utilizzare l'overload che accetta un <xref:System.IFormatProvider> argomento. In alternativa, usare una [stringa interpolata](/dotnet/csharp/tutorials/string-interpolation) <xref:System.FormattableString.Invariant%2A?displayProperty=nameWithType> in C, e passarla al metodo.

## <a name="when-to-suppress-warnings"></a>Quando eliminare gli avvisi

È possibile escludere un avviso da questa regola quando è certo che il formato predefinito è la scelta corretta e in cui la manutenibilità del codice non è una priorità di sviluppo importante.

## <a name="example"></a>Esempio

Nel codice seguente, `example1` la stringa viola la regola CA1305. La `example2` stringa soddisfa la regola CA1305 <xref:System.Globalization.CultureInfo.CurrentCulture%2A?displayProperty=nameWithType>passando , <xref:System.IFormatProvider>che <xref:System.String.Format(System.IFormatProvider,System.String,System.Object)?displayProperty=nameWithType>implementa , a . La `example3` stringa soddisfa la regola CA1305 passando una stringa <xref:System.FormattableString.Invariant%2A?displayProperty=fullName]>interpolata a .

```csharp
string name = "Georgette";

// Violates CA1305
string example1 = String.Format("Hello {0}", name);

// Satisfies CA1305
string example2 = String.Format(CultureInfo.CurrentCulture, "Hello {0}", name);

// Satisfies CA1305
string example3 = FormattableString.Invariant($"Hello {name}");
```
