# <a name="ca1303-do-not-pass-literals-as-localized-parameters"></a>CA1303: Non passare valori letterali come parametri localizzati

|||
|-|-|
|TypeName|DoNotPassLiteralsAsLocalizedParameters|
|CheckId|CA1303|
|Category|Microsoft.Globalization (Globalizzazione)|
|Modifica|Non-rottura|

## <a name="cause"></a>Causa

Un metodo passa un valore letterale stringa come parametro a un costruttore o a un metodo .NET e tale stringa deve essere localizzabile.

Questo avviso viene generato quando una stringa letterale viene passata come valore a un parametro o a una proprietà e uno o più dei seguenti casi è true:This warning is raised when a literal string is passed as a value to a parameter or property and one or more of the following cases is true:

- L'attributo <xref:System.ComponentModel.LocalizableAttribute> del parametro o della proprietà è impostato su true.

- Il nome del parametro di stringa passato a un metodo Console.Write o Console.WriteLine è "value" o "format".

Per impostazione predefinita, questa regola analizza l'intera codebase, ma è [configurabile.](#excluded-type-names-with-derived-types)

A partire dalla versione 3.0.0 del pacchetto analizzatore, l'euristica di denominazione non viene più utilizzata per impostazione predefinita, ma è [configurabile.](#use-naming-heuristic)

## <a name="rule-description"></a>Descrizione della regola

I valori letterali stringa incorporati nel codice sorgente sono difficili da localizzare.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

Per correggere una violazione di questa regola, sostituire il valore <xref:System.Resources.ResourceManager> letterale stringa con una stringa recuperata tramite un'istanza della classe.

## <a name="when-to-suppress-warnings"></a>Quando eliminare gli avvisi

È possibile escludere un avviso da questa regola se la libreria di codice non verrà localizzata o se la stringa non è esposta all'utente finale o a uno sviluppatore che utilizza la libreria di codice.

Gli utenti possono eliminare il rumore rispetto ai metodi che non devono essere passati stringhe localizzate rinominando il parametro o la proprietà o contrassegnando questi elementi come condizionali.

## <a name="configurability"></a>Configurabilità

Se si esegue questa regola dagli [analizzatori FxCop](install-fxcop-analyzers.md) (e non con l'analisi legacy), questa regola è configurabile.

### <a name="excluded-type-names-with-derived-types"></a>Nomi di tipo esclusi con tipi derivati

È possibile configurare i tipi, inclusi i tipi derivati, da escludere dall'analisi. Ad esempio, per specificare che la regola non `MyType` deve essere eseguita su tutti i metodi all'interno di tipi denominati e dei relativi tipi derivati, aggiungere la seguente coppia chiave-valore a un file con estensione editorconfig nel progetto:

```ini
dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType
```

Formati di nomi di simboli consentiti nel valore dell'opzione (separati da ''):Allowed symbol name formats in the option value (separated by '''):

- Solo nome del tipo (include tutti i tipi con il nome, indipendentemente dal tipo o dallo spazio dei nomi che lo contiene)
- Nomi completi nel formato [dell'ID di documentazione](https://github.com/dotnet/csharplang/blob/master/spec/documentation-comments.md#id-string-format)del simbolo, con un prefisso facoltativo. `T:`

Esempi:

| Valore dell'opzione | Summary |
| --- | --- |
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType` | Corrisponde a tutti i tipi denominati 'MyType' e a tutti i relativi tipi derivati nella compilazione
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = MyType1|MyType2` | Corrisponde a tutti i tipi denominati 'MyType1' o 'MyType2' e a tutti i relativi tipi derivati nella compilazione
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = M:NS.MyType` | Corrisponde al tipo specifico 'MyType' con il nome completo specificato e tutti i relativi tipi derivati
|`dotnet_code_quality.CA1303.excluded_type_names_with_derived_types = M:NS1.MyType1|M:NS2.MyType2` | Corrisponde a tipi specifici 'MyType1' e 'MyType2' con i rispettivi nomi completi e tutti i relativi tipi derivati

È possibile configurare questa opzione solo per questa regola, per tutte le regole o per tutte le regole di questa categoria (Globalizzazione). Per ulteriori informazioni, consultate [Configurare gli analizzatori FxCop.](configure-fxcop-analyzers.md)

### <a name="use-naming-heuristic"></a>Usare l'euristica di denominazioneUse naming heuristic

È possibile configurare se i parametri o il nome delle proprietà contenenti "Text", "Message" o "Caption" attiverà questa regola.

```ini
dotnet_code_quality.CA1303.use_naming_heuristic = true
```
