# <a name="ca3061-do-not-add-schema-by-url"></a>CA3061: Non aggiungere lo schema in base all'URL

|||
|-|-|
|CheckId|CA3061|
|Category|Microsoft.Security|
|Modifica|Senza interruzioni|

## <a name="cause"></a>Causa

L'overload di `XmlSchemaCollection.Add(String, String)` utilizza `XmlUrlResolver` per specificare XML schema esterno sotto forma di un URI. Se la stringa URI è viziata, può causare l'analisi di una XML Schema dannosa, che consente l'inclusione di bombe XML e entità esterne dannose. Questo potrebbe consentire a un utente malintenzionato malintenzionato di eseguire un attacco di tipo Denial of Service, divulgazione di informazioni o richiesta lato server.

## <a name="rule-description"></a>Descrizione della regola

Non usare l'overload unsafe del `Add` metodo perché potrebbe causare riferimenti esterni pericolosi.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

- Non usare `XmlSchemaCollection.Add(String, String)`.

## <a name="when-to-suppress-warnings"></a>Quando escludere gli avvisi

Eliminare questa regola se si è certi che il codice XML non risolve i riferimenti esterni pericolosi.

## <a name="pseudo-code-examples"></a>Esempi di pseudo-codice

### <a name="violation"></a>Violazione

Nell'esempio di pseudo-codice seguente viene illustrato il modello rilevato da questa regola.
Il tipo del secondo parametro è `string` .

```csharp
using System;
using System.Xml.Schema;
...
XmlSchemaCollection xsc = new XmlSchemaCollection();
xsc.Add("urn: bookstore - schema", "books.xsd");
```

### <a name="solution"></a>Soluzione

```csharp
using System;
using System.IO;
using System.Xml;
using System.Xml.Schema;
...
XmlSchemaCollection xsc = new XmlSchemaCollection();
xsc.Add("urn: bookstore - schema", new XmlTextReader(new FileStream(""xmlFilename"", FileMode.Open)));
```
