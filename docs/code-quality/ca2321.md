# <a name="ca2321-do-not-deserialize-with-javascriptserializer-using-a-simpletyperesolver"></a>CA2321: non eseguire la deserializzazione con JavaScriptSerializer usando un SimpleTypeResolver

|||
|-|-|
|TypeName|DoNotDeserializeWithJavaScriptSerializerUsingASimpleTypeResolver|
|CheckId|CA2321|
|Categoria|Microsoft.Security|
|Modifica|Senza interruzioni|

## <a name="cause"></a>Causa

Ãˆ stato chiamato o fatto riferimento a un metodo di deserializzazione <xref:System.Web.Script.Serialization.JavaScriptSerializer?displayProperty=nameWithType> dopo l'inizializzazione con una <xref:System.Web.Script.Serialization.SimpleTypeResolver?displayProperty=nameWithType>.

## <a name="rule-description"></a>Descrizione della regola

[!INCLUDE[insecure-deserializers-description](includes/insecure-deserializers-description-md.md)]

Questa regola trova <xref:System.Web.Script.Serialization.JavaScriptSerializer?displayProperty=nameWithType> le chiamate al metodo di deserializzazione o i riferimenti, dopo l'inizializzazione del <xref:System.Web.Script.Serialization.JavaScriptSerializer> con un <xref:System.Web.Script.Serialization.SimpleTypeResolver?displayProperty=nameWithType>.

## <a name="how-to-fix-violations"></a>Come correggere le violazioni

- Non inizializzare <xref:System.Web.Script.Serialization.JavaScriptTypeResolver?displayPropertyName=nameWithType> con una <xref:System.Web.Script.Serialization.SimpleTypeResolver?displayProperty=nameWithType>.
- Se il codice deve leggere i dati serializzati utilizzando una <xref:System.Web.Script.Serialization.SimpleTypeResolver>, limitare i tipi deserializzati a un elenco previsto implementando una <xref:System.Web.Script.Serialization.JavaScriptTypeResolver>personalizzata.
- Eseguire la prova di manomissione dei dati serializzati. Dopo la serializzazione, la firma crittografica dei dati serializzati. Prima della deserializzazione, convalidare la firma crittografica. Proteggere la chiave crittografica da divulgare e progettare per le rotazioni delle chiavi.

### <a name="solution"></a>Soluzione

```csharp
using System;
using System.Web.Script.Serialization;

public class BookRecordTypeResolver : JavaScriptTypeResolver
{
    // For compatibility with data serialized with a JavaScriptSerializer initialized with SimpleTypeResolver.
    private static readonly SimpleTypeResolver Simple = new SimpleTypeResolver();

    public override Type ResolveType(string id)
    {
        // One way to discover expected types is through testing deserialization
        // of **valid** data and logging the types used.

        ////Console.WriteLine($"ResolveType('{id}')");

        if (id == typeof(BookRecord).AssemblyQualifiedName || id == typeof(AisleLocation).AssemblyQualifiedName)
        {
            return Simple.ResolveType(id);
        }
        else
        {
            throw new ArgumentException("Unexpected type ID", nameof(id));
        }
    }

    public override string ResolveTypeId(Type type)
    {
        return Simple.ResolveTypeId(type);
    }
}

public class BookRecord
{
    public string Title { get; set; }
    public string Author { get; set; }
    public int PageCount { get; set; }
    public AisleLocation Location { get; set; }
}

public class AisleLocation
{
    public char Aisle { get; set; }
    public byte Shelf { get; set; }
}

public class ExampleClass
{
    public BookRecord DeserializeBookRecord(string s)
    {
        JavaScriptSerializer serializer = new JavaScriptSerializer(new BookRecordTypeResolver());
        return serializer.Deserialize<BookRecord>(s);
    }
}
```
