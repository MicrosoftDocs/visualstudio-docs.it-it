# <a name="c6387"></a>C6387
avviso C6387: \<argument > può essere \<value >: non rispetta la specifica per la funzione \<function nome >: righe: x, y

 Questo avviso viene generato se al parametro di una funzione con annotazioni viene passato un valore imprevisto. Se ad esempio si passa un valore potenzialmente null a un parametro contrassegnato con l'annotazione `_In_`, viene generato questo avviso.

## <a name="example"></a>Esempio
 Il codice seguente genera questo avviso perché un parametro null viene passato a `f(char *)`:

```cpp

#include <sal.h>

_Post_ _Null_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh); // Warning C6387
}
```

 Per correggere il problema, usare il codice seguente:

```cpp

#include <sal.h>

_Post_ _Notnull_ char * g();

void f(_In_ char *pch);

void main()
{
    char *pCh = g();
    f(pCh);
}
```
