# <a name="c6284"></a>C6284
avviso C6284: oggetto passato come parametro '% d'quando è richiesta una stringa nella chiamata a \<function >.

 Questo avviso indica che la stringa di formato specifica una stringa, ad esempio una specifica `%s` per `printf` o `scanf`, ma è C++ stato invece passato un oggetto.

 Questo difetto potrebbe produrre un output errato o arresti anomali.

 Questo messaggio viene spesso segnalato a causa del passaggio C++ di un oggetto che implementa un tipo stringa, ad esempio `std::string`, `CComBSTR` o `bstr_t`, in una chiamata in stile C `printf`. A seconda dell'implementazione della C++ classe, ovvero se sono definiti gli operatori cast appropriati, C++ gli oggetti stringa possono spesso essere utilizzati in modo trasparente ogni volta che sono richieste stringhe C. Tuttavia, poiché i parametri per le funzioni di tipo `printf` sono essenzialmente non tipizzati, non viene eseguita alcuna conversione in una stringa.

 A seconda dell'oggetto, potrebbe essere opportuno inserire un operatore `static_cast` al tipo di stringa appropriato, ad esempio `char *` o `TCHAR``*` oppure chiamare una funzione membro che restituisce una stringa, ad esempio `c_str()`, nelle istanze di `std::string`.

## <a name="example"></a>Esempio
 Il codice seguente genera questo avviso perché un `CComBSTR` viene passato alla funzione `sprintf`:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 Il codice seguente usa il cast statico per correggere questo avviso:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
