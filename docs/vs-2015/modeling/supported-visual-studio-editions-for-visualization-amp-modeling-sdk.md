# <a name="supported-visual-studio-editions-for-visualization-amp-modeling-sdk"></a>Le edizioni di Visual Studio supportate per la visualizzazione &amp; Modeling SDK
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

La versione più recente di questo argomento è reperibile in [supportate le edizioni Visual Studio per la visualizzazione &amp; Modeling SDK](https://docs.microsoft.com/visualstudio/modeling/supported-visual-studio-editions-for-visualization-amp-modeling-sdk).  
  
Di seguito sono riportati gli elenchi delle versioni di Visual Studio che sono supportate con [!INCLUDE[dsl](../includes/dsl-md.md)] negli ambienti di creazione e distribuzione. Per altre informazioni su queste edizioni, vedere Microsoft [!INCLUDE[vsprvs](../includes/vsprvs-md.md)] [Centro per sviluppatori](http://go.microsoft.com/fwlink/?LinkId=75628).  
  
## <a name="authoring-edition"></a>Edizione per lo sviluppo

## <a name="deployment-editions"></a>Edizioni per la distribuzione  
 [!INCLUDE[dsl](../includes/dsl-md.md)] supporta le configurazioni seguenti per la distribuzione dei linguaggi specifici di dominio compilati:  
  
-   Visual Studio Enterprise  
  
-   Visual Studio Professional  
  
-   Visual Studio Shell (modalità integrata) pacchetto ridistribuibile  
  
-   Pacchetto ridistribuibile Visual Studio Shell (modalità isolata)  
  
> [!NOTE]
>  Per rendere un linguaggio DSL può essere eseguita su un prodotto Shell, è necessario impostare il **edizione supportata di Visual Studio** campo nel manifesto dell'estensione. Per altre informazioni, vedere [distribuzione di soluzioni Domain-Specific Language](../modeling/deploying-domain-specific-language-solutions.md).
