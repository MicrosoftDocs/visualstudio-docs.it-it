# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

avviso C6221: cast implicito tra tipi integer semanticamente diversi: confronto di HRESULT con un Integer. Si consiglia di utilizzare macro SUCCEEDed o FAILED  
  
 Questo avviso indica che un `HRESULT` viene confrontato con un valore integer diverso da zero. Una riuscita in `HRESULT` (`S_OK`) è rappresentata da un valore 0. Pertanto, un cast implicito di un `HRESULT` a un intero genererà un valore non corretto e probabilmente genererà un risultato errato. Spesso viene causata erroneamente che una funzione restituisca un valore integer quando restituisce effettivamente un `HRESULT`.  
  
## <a name="example"></a>Esempio  
 Il codice seguente genera questo avviso confrontando `HRESULT` rispetto a un valore integer:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Per correggere il problema, il codice seguente usa la macro `FAILED`:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Per questo avviso, il tipo di `SCODE` è equivalente a `HRESULT`.  
  
 Per ulteriori informazioni, vedere [macro SUCCEEDED](https://go.microsoft.com/fwlink/?LinkId=92738) e [macro failed](https://go.microsoft.com/fwlink/?LinkId=180875).  
  
 Si noti che l'uso di malloc e della versione gratuita (e delle API di allocazione della memoria dinamica correlate) presenta molti problemi in termini di perdite di memoria ed eccezioni. Per evitare completamente questi tipi di perdite e problemi di eccezione, utilizzare i meccanismi forniti dalla libreria STL ( C++ Standard Template Library). Per altre informazioni, vedere [puntatori intelligenti](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) e [ C++ libreria standard](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
