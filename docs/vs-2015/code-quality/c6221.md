# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

avviso C6221: cast implicito tra tipi integer semanticamente diversi: confronto di HRESULT con un numero intero. Prendere in considerazione l'utilizzo della macro SUCCEEDED o FAILED  
  
 Questo avviso indica che un `HRESULT` viene confrontato a un valore integer diverso da zero. L'esito positivo in `HRESULT` (`S_OK`) è rappresentato da un valore 0. Pertanto, un cast implicito di un `HRESULT` in un numero intero, verrà generato un valore non corretto ed è probabile che produce il risultato non corretto. Ciò è spesso causato dalla erroneamente prevede una funzione per restituire un valore intero quando in realtà restituisce un `HRESULT`.  
  
## <a name="example"></a>Esempio  
 Il codice seguente genera l'avviso confrontando `HRESULT` rispetto a un valore integer:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Per risolvere questo problema, il codice seguente usa il `FAILED` macro:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Per questo avviso, il `SCODE` il tipo è equivalente a `HRESULT`.  
  
 Per altre informazioni, vedere [Macro SUCCEEDED](http://go.microsoft.com/fwlink/?LinkId=92738) e [Macro FAILED](http://go.microsoft.com/fwlink/?LinkId=180875).  
  
 Si noti che l'uso di malloc e free (e correlate allocazione dinamica della memoria API) hanno molte insidie in termini di perdite di memoria e le eccezioni. Per evitare questi tipi di eccezione e le perdite di problemi di inconsistenze, utilizzano i meccanismi forniti dalla libreria di modello Standard (STL) di C++. Questi includono [shared_ptr](http://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](http://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a), e [vettore](http://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Per altre informazioni, vedere [puntatori intelligenti](http://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) e [libreria Standard C++](http://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
