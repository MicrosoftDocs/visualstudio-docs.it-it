# <a name="c6216"></a>C6216
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

avviso C6216: cast inserito dal compilatore tra tipi integrali semanticamente diversi: da un tipo Boolean a HRESULT  
  
 Questo avviso indica che un valore booleano viene usato come oggetto `HRESULT` senza che venga eseguito il cast esplicito. I tipi booleani indicano l'esito positivo di un valore diverso da zero. Success ( `S_OK` ) in `HRESULT` è indicato da un valore pari a 0. Il valore di errore tipico per le funzioni che restituiscono un valore booleano false è uno stato di esito positivo quando viene testato come `HRESULT` . È probabile che si verifichino risultati non corretti.  
  
## <a name="example"></a>Esempio  
 Il codice seguente genera questo avviso:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
HRESULT f( REFGUID riid1, REFGUID riid2 )  
{  
  // code ...  
  return IsEqual(riid1, riid2);    
}  
```  
  
 Per correggere il problema, usare il codice seguente:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
HRESULT f( REFGUID riid1, REFGUID riid2 )  
{  
  if (IsEqual(riid1, riid2) == TRUE)  
  {  
    // code ...  
    return S_OK;  
  }  
  else  
  {  
    // code ...  
    return E_FAIL;  
  }  
}  
```  
  
 Per questo avviso, il `SCODE` tipo è equivalente a `HRESULT` .  
  
 Per ulteriori informazioni, vedere [macro SUCCEEDED](https://msdn.microsoft.com/library/ms687197.aspx) e [macro failed](https://msdn.microsoft.com/library/ms693474.aspx).
