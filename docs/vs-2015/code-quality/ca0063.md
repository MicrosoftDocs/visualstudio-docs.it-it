# <a name="ca0063"></a>CA0063

Non è stato possibile caricare il file del set di regole o uno dei file del set di regole dipendenti.

Non è stato possibile caricare il file del set di regole o uno dei file del set di regole dipendenti.

Il set di regole specificato non è stato trovato oppure non è stato possibile trovare uno dei file di RuleSet inclusi nel set di regole. Assicurarsi che tutti i RuleSet inclusi nel set di regole esistano sul disco e che nel progetto siano specificate le directory di RuleSet appropriate tramite la proprietà **CodeAnalysisRuleSetDirectories** di MSBuild.

Per eseguire il debug dell'errore, esaminare il file di RuleSet in un editor di testo. Per trovare il percorso del file di RuleSet, fare clic con il pulsante destro del mouse sul progetto in Esplora soluzioni, scegliere **Proprietà**, quindi fare clic su **analisi del codice**. Verificare che il file di RuleSet sia selezionato in **Esegui questo set di regole**. Il percorso del set di regole è elencato nel campo Descrizione.

Esaminare il valore dell'attributo **path** per tutti gli elementi **include** . I percorsi di inclusione possono usare percorsi relativi per il file di RuleSet padre/corrente, le variabili di ambiente e i percorsi assoluti. Esempio:

```
<Include Path="%PUBLIC%\Documents\RuleSets\alldesign.ruleset" Action="Default" />
<Include Path="..\alldesign.ruleset" Action="Default" />
<Include Path="C:\Rulesets\alldesign.ruleset" Action="Default" />
```

Controllare ognuno di questi percorsi di inclusione e verificare che siano tutti validi.

In alcuni casi, la posizione di un set di regole può dipendere da una proprietà di MSBuild. Non è possibile fare riferimento alle proprietà MSBuild da un set di regole. Per risolvere questo problema, specificare percorsi di ricerca aggiuntivi nella proprietà **CodeAnalysisRuleSetDirectories** di MSBuild. In questo scenario, specificare solo il nome del set di regole nell'attributo **path** dell'elemento **include** .
