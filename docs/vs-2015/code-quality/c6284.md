# <a name="c6284"></a>C6284
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

avviso C6284: oggetto passato come parametro '% d'quando è richiesta una stringa nella chiamata a \<function> .  
  
 Questo avviso indica che la stringa di formato specifica una stringa, ad esempio una `%s` specifica per `printf` o `scanf` , ma è stato invece passato un oggetto C++.  
  
 Questo difetto potrebbe produrre un output errato o arresti anomali.  
  
 Questo messaggio viene spesso segnalato a causa del passaggio di un oggetto C++ che implementa un tipo stringa, ad esempio, `std::string` `CComBSTR` o `bstr_t` , in una chiamata di tipo C `printf` . A seconda dell'implementazione della classe C++, ovvero se sono definiti gli operatori cast appropriati, gli oggetti stringa C++ possono spesso essere utilizzati in modo trasparente ogni volta che sono richieste stringhe C. Tuttavia, poiché i parametri per le `printf` funzioni di tipo sono essenzialmente non tipizzati, non viene eseguita alcuna conversione in una stringa.  
  
 A seconda dell'oggetto, potrebbe essere opportuno inserire un `static_cast` operatore nel tipo di stringa appropriato, ad esempio `char *` o `TCHAR``*` , oppure per chiamare una funzione membro che restituisce una stringa, ad esempio, `c_str()` nelle istanze di `std::string` .  
  
## <a name="example"></a>Esempio  
 Il codice seguente genera questo avviso perché un oggetto `CComBSTR` viene passato alla `sprintf` funzione:  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf(buff,"%ws",bstrValue);   
}  
```  
  
 Il codice seguente usa il cast statico per correggere questo avviso:  
  
```  
#include <atlbase.h>  
#include <stdlib.h>  
  
void f()  
{  
  char buff[50];  
  CComBSTR bstrValue("Bye");  
  
  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));  
}  
```
