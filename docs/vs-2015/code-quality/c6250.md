# <a name="c6250"></a>C6250
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

avviso C6250: \<function> la chiamata a VirtualFree senza il flag di MEM_RELEASE può liberare memoria ma non descrittori di indirizzo (VADS); genera perdite di spazio degli indirizzi  
  
 Questo avviso indica che una chiamata a `VirtualFree` senza il `MEM_RELEASE` flag Annulla solo il commit delle pagine e non le rilascia. Per eseguire il decommit e rilasciare le pagine, usare `MEM_RELEASE` flag nella chiamata a `VirtualFree` . Se viene eseguito il commit di tutte le pagine dell'area, la funzione esegue prima di tutto il commit e quindi le rilascia. Dopo questa operazione, le pagine sono in stato libero. Se si specifica questo flag, `dwSize` deve essere zero e `lpAddress` deve puntare all'indirizzo di base restituito dalla `VirtualAlloc` funzione quando l'area è stata riservata. La funzione ha esito negativo se una di queste condizioni non viene soddisfatta.  
  
 È possibile ignorare questo avviso se il codice libera in un secondo momento lo spazio degli indirizzi chiamando `VirtualFree` con il `MEM_RELEASE` flag.  
  
 Per ulteriori informazioni, vedere [VirtualAlloc](https://msdn.microsoft.com/library/aa366887.aspx) e [VirtualFree](https://msdn.microsoft.com/library/aa366892.aspx).  
  
 L'uso di VirtualAlloc e VirtualFree presenta molti problemi in termini di perdite di memoria ed eccezioni. Per evitare completamente questi tipi di perdite e problemi di eccezione, utilizzare i meccanismi forniti dalla libreria STL (Standard Template Library) di C++. Sono inclusi [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)e [vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Per altre informazioni, vedere [puntatori intelligenti](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) e [libreria standard C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).  
  
## <a name="example"></a>Esempio  
 Il codice di esempio seguente genera questo avviso:  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages  
      {  
        puts ("MEM_DECOMMIT Succeeded");  
      }  
      else  
      {  
       puts("MEM_DECOMMIT failed");  
      }  
  }  
  else  
  {  
    puts("lpvBase == NULL");  
  }  
}  
```  
  
 Per correggere il problema, usare il codice di esempio seguente:  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
   if (VirtualFree(lpvBase, 0,MEM_RELEASE )) // decommit & release   
   {  
     // code ...  
   }  
   else  
   {  
     puts("MEM_RELEASE failed");  
   }  
  }  
  else   
  {  
    puts("lpvBase == Null ");  
    // code...  
  }  
}  
```
