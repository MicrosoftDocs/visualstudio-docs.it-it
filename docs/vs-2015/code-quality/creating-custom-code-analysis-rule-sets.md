# <a name="creating-custom-code-analysis-rule-sets"></a>Creazione di set di regole di analisi del codice personalizzati
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Per applicare regole di analisi del codice a un progetto di codice gestito, è necessario specificare un set di regole che contenga una o più regole. I set di regole standard Microsoft forniscono gruppi di regole organizzati per funzione e profondità. Ad esempio, le regole delle linee guida di progettazione di base di Microsoft e le regole delle linee guida di progettazione estese Microsoft contengono regole che si concentrano sui problemi di usabilità e gestibilità, con maggiore enfasi sulle regole di denominazione nel set di regole estese.

 È inoltre possibile creare un set di regole personalizzato che combina due o più set di regole standard o che specifica le regole in uno o più set di regole standard. I set di regole personalizzati vengono salvati in un file con estensione ruleset ed è possibile specificarli in più progetti di codice.
