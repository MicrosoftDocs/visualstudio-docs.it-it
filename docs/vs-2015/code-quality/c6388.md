# <a name="c6388"></a>C6388
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

avviso C6388: \<argomento > non può essere \<valore >: non rispetta la specifica per la funzione \<nome della funzione >: righe: x, y  
  
 Questo avviso indica che è in uso un valore imprevisto nel contesto specificato. Questa operazione viene in genere segnalata per i valori passati come argomenti a una funzione che non lo prevede.  
  
## <a name="example"></a>Esempio  
 Il codice C++ seguente genera questo avviso perché DoSomething prevede un valore null, ma potrebbe essere passato un valore potenzialmente non null:  
  
```cpp  
  
#include <string.h>  
#include <malloc.h>  
#include <sal.h>  
  
void DoSomething( _Pre_ _Null_ void* pReserved );  
  
void f()  
{  
    void* p = malloc( 10 );  
    DoSomething( p );  // Warning C6388  
    // code...  
    free(p);  
}  
```  
  
 Per correggere il problema, usare il codice di esempio seguente:  
  
```cpp  
  
#include <string.h>  
#include <malloc.h>  
#include <sal.h>  
  
void DoSomething( _Pre_ _Null_ void* pReserved );  
void f()  
{  
    void* p = malloc( 10 );  
    if (!p)  
    {  
        DoSomething( p );    
    }  
    else  
    {  
        // code...  
        free(p);  
    }  
}  
```  
  
 Si noti che l'uso di malloc e Free presenta molti problemi in termini di perdite di memoria ed eccezioni. Per evitare completamente questi tipi di perdite e problemi di eccezione, utilizzare i meccanismi forniti dalla libreria STL ( C++ Standard Template Library). Sono inclusi [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)e [vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Per altre informazioni, vedere [puntatori intelligenti](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) e [ C++ libreria standard](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
