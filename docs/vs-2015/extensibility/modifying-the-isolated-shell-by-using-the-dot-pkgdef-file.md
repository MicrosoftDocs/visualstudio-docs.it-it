# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Modifica della shell isolata tramite il file con estensione pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Il file con estensione pkgdef supporta le impostazioni che è possibile usare per personalizzare un'applicazione shell isolata. Specifica i valori creati quando un'applicazione viene installata in un computer e a cui fa riferimento la shell di Visual Studio all'avvio dell'applicazione. Le impostazioni sono organizzate nel file in base alle chiavi del registro di sistema applicabili.  

> [!WARNING]
> Si noti che i file con estensione pkgdef che non sono dichiarati nel file con estensione vsixmanifest del pacchetto VSPackage non vengono analizzati all'avvio di Visual Studio.  

 Il file. pkgdef contiene sezioni che sono ognuna identificata da una chiave, `[$RootKey$]` o `[$RootKey$\` *sottochiave* `]` , dove $RootKey $ è la chiave radice per l'applicazione.  

 Ogni sezione contiene coppie nome/valore con il formato seguente: `"` *valueName* `"=` *Value*.  

 I valori sono una stringa racchiusa tra virgolette o un intero a 32 bit rappresentato come un valore DWORD. I valori presentano i vincoli seguenti:  

- Tutti i valori DWORD sono in formato esadecimale, ad esempio `dword:00000001` .  

   Per i valori booleani, 1 rappresenta true e 0 rappresenta false.  

- Tutte le stringhe GUID sono nel formato del registro di sistema, ad esempio `"{00000000-0000-0000-0000-000000000000}"` .  

- Tutti gli identificatori di risorsa localizzabili hanno il formato "@*resourceId*" o "#*resourceId*", dove *resourceId* è l'identificatore di risorsa nel pacchetto dell'interfaccia utente dell'applicazione, ad esempio `"@102"` . Il pacchetto dell'interfaccia utente dell'applicazione è il pacchetto a cui si fa riferimento nell'impostazione AppLocalizationPackage.  

  Ad esempio:  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 È possibile aggiungere commenti al file con estensione pkgdef. Un commento a riga singola è costituito da due barre come i primi due caratteri.  

 Per un elenco delle stringhe di sostituzione, vedere [stringhe di sostituzione utilizzate in. Pkgdef e. File Pkgundef](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 Le sezioni seguenti descrivono valori specifici del registro di sistema che influiscono sul comportamento della shell di Visual Studio in modalità isolata. È anche possibile definire altri valori del registro di sistema per l'applicazione in questo file.  

> [!NOTE]
> Se non viene specificata un'impostazione nel file con estensione pkgdef, non viene eseguita alcuna voce corrispondente nel registro di sistema.

## <a name="binding-path-settings"></a>Impostazioni del percorso di associazione  
 La chiave [$RootKey $ \BindingPaths \\ {00000000-0000-0000-0000-000000000000} ] contiene l'elenco delle directory che la shell verifica per gli assembly. Queste directory vengono aggiunte all'elenco di directory in cui la shell esegue il probe degli assembly privati per l'applicazione.  

 Per impostazione predefinita, non viene aggiunta alcuna voce relativa al percorso di associazione al file con estensione pkgdef. Tuttavia, le seguenti sottodirectory della directory di installazione di Visual Studio vengono aggiunte automaticamente all'elenco di percorsi di binding dell'applicazione nel registro di sistema.  

- Common7\IDE\  

- \\\PrivateAssemblies Common7\IDE  

- \\\PublicAssemblies Common7\IDE  

  Per aggiungere una directory al percorso di associazione, aggiungere una voce nel formato "*DirectoryName*" = "", dove *DirectoryName* è un percorso assoluto. Ad esempio:  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>Impostazioni delle voci di menu del pacchetto  
 La chiave del registro di sistema [$RootKey $ \Menus] definisce i file di risorse dell'interfaccia utente per l'applicazione.  

 I valori delle voci di menu hanno il formato "{*vsUiPackageGuid*}" = ", *resourceId*, *VersionNumber*", dove *vsUiPackageGuid* è il GUID del pacchetto dell'interfaccia utente dell'applicazione, *resourceId* è l'identificatore di risorsa della risorsa CTMENU che contiene gli elementi dell'interfaccia utente e *VersionNumber* è un numero di versione virtuale per la risorsa CTMENU. Per ulteriori informazioni, vedere la pagina relativa alla [registrazione dei gestori di comandi di assembly di interoperabilità](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 Per impostazione predefinita, viene creata una voce di voce di menu nel file con estensione pkgdef per il pacchetto dell'interfaccia utente dell'applicazione.  

 Per ogni pacchetto che fornisce voci di menu e distribuite come parte dell'applicazione, aggiungere una voce di menu per il pacchetto.
