# <a name="vs-connected-environment-for-azure-container-service-aks-preview"></a>Visual Studio Connected Environment per il servizio contenitore di Azure (anteprima)
Visual Studio Connected Environment consente di sviluppare con velocità in Kubernetes. È possibile creare un ambiente basato su Kubernetes completamente gestito in Azure e ottimizzato per lo sviluppo, quindi sviluppare in modo iterativo contenitori nel cloud usando strumenti familiari come Visual Studio Code, Visual Studio o la riga di comando.

Questo approccio offre diversi vantaggi:

* Ottenere un ambiente di sviluppo senza infrastruttura rappresentativo dell'ambiente di produzione, con accesso completo alle risorse cloud.
* Eseguire il debug dei contenitori Node.js e .NET Core direttamente in Kubernetes con Visual Studio Code o Visual Studio. Lo sviluppo con tutti gli altri linguaggi può essere gestito con l'interfaccia della riga di comando.
* Condividere un'istanza di Kubernetes tra il team di sviluppo per ridurre i costi e ridurre al minimo l'installazione di computer locali per i nuovi membri del team.
* Sviluppare il codice in isolamento ed eseguire test completi con altri componenti senza replicare o simulare le dipendenze.
